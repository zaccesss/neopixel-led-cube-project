# Documentation

This directory contains all project documentation including market analysis, financial reports, project management materials and technical portfolios. These documents provide context and supporting information for the NeoPixel LED Cube Project.

## Document Overview

### market-analysis.docx

**Purpose:** Competitive market analysis of similar LED cube products

**Contents:**
- Comparison of competing commercial LED cube kits
- Price points and feature sets
- Analysis of available products on Amazon and AliExpress
- Identification of market gaps and opportunities

**Key Findings:**
The market analysis identified three primary competing products ranging from £21.99 to £36.00. Commercial offerings typically provide pre-assembled solutions with varying LED counts and feature sets. This project differentiates itself through academic focus, custom design and comprehensive documentation.

### finance-report.xlsx

**Purpose:** Project budget tracking and financial documentation

**Contents:**
- Component costs and sourcing information
- Budget allocation across hardware, materials and tools
- Cost comparison with commercial alternatives
- Return on investment analysis for educational applications

**Summary:**
The spreadsheet provides detailed financial tracking for the project, demonstrating cost-effective implementation compared to commercial alternatives while delivering superior educational value through hands-on construction and programming experience.

### gantt-chart.xlsx

**Purpose:** Project timeline and task management

**Contents:**
- Project phases and milestones
- Task dependencies and critical path
- Resource allocation timeline
- Progress tracking against planned schedule

**Project Phases:**
1. Planning and design (component selection, circuit design)
2. Hardware assembly (cube construction, enclosure fabrication)
3. Software development (pattern programming, sensor integration)
4. Testing and refinement (debugging, optimization)
5. Documentation (technical writing, media production)

### hardware-overview.pptx

**Purpose:** Visual presentation of hardware design and assembly

**Contents:**
- Component specifications and rationale
- Circuit diagrams and wiring schematics
- Assembly process photography
- Design decisions and engineering trade-offs

**Target Audience:**
This presentation serves as a technical overview for academic evaluation and provides visual documentation of the construction process for future reference or replication.

### notes-and-drafts.docx

**Purpose:** Early planning documents and development notes

**Contents:**
- Initial project requirements (essential and recommended features)
- Hardware selection criteria
- Early code snippets and algorithm concepts
- Design iteration notes

**Historical Value:**
This document preserves the project's evolution from initial concept to final implementation, showing design decisions and problem-solving approaches throughout development.

### software-portfolio.docx

**Purpose:** Comprehensive code documentation and software architecture overview

**Contents:**
- Annotated code screenshots with detailed explanations
- Function-by-function breakdown of software logic
- Software flowchart illustrating program execution
- Animation pattern descriptions

**Key Sections:**

**Documentation Style:**
The portfolio uses visual aids (screenshots and diagrams) combined with descriptive text to explain software functionality. Each code section includes both technical explanation and practical context.
